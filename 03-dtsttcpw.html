DTSTTCPW

Do The Simplest Thing That Could Possibly Work
Het principe dat ontwikkelaars zich moeten richten op huidige vereisten en niet moet speculeren op toekomstige behoeften. Onderdeel van DTSTTCPW ().

  1. Focus op eenvoud
  2. Iteratieve verbeteringen
  3. Vermijd speculatie
  4. Snelle feedback
  5. Minimalistisch ontwerp

Complexity is your enemy. Any fool can make something complicated. It is hard to keep things simple.
~ Richard Branson - Muziek en vliegtuig meneer

Focus op eenvoud

Bij het oplossen van een programmeer vraagstuk probeer de focus te leggen op eenvoud. Niet de korste of makkelijkste code maar de gemakkelijkst te begrijpen en te onderhouden. Probeer de code te zien vanuit het perspectief van je college of je toekomstige zelf. Kun je de code in een oogopslag doorgronden? Begrijp je wat hier gebeurt?

Iteratieve verbeteringen / refactor

Verbeter gaandeweg je code. Probeer niet in één keer de code perfect uit te schrijven.
Een werkwijze zou kunnen zijn:

  • Splits functionele opdracht uit in losse taken
  • Schrijf pseudo code
  • Schrijf functies
  • Refactor functies
  • Schrijf unit testen
  • Refactor functies

Vermijd speculatie

Voor het uitprogammeren van een PBI of Bug zal een functionele beschrijving beschikbaar zijn. Het kan voorkomen dat deze niet volledig de lading dekt. Het is eenvoudig om zelf een draai te geven aan de requirements.
Om te vermijden dat tijd wordt verspild aan taken die niet expliciet gevraagd zijn (scope-creep), of dat uiteindelijk het verkeerde werk wordt verricht, is het raadzaam enkele minuten te besteden aan het opstellen van een beknopte beschrijving van de geplande werkzaamheden. Bespreek dit vervolgens met de opdrachtgever om zeker te zijn dat je op één lijn zit met hun verwachtingen.

Snelle feedback

Soms krijg je iets niet voor elkaar om verscheidenheid van redenen. Het is natuurlijk prima om op zoek te gaan om het vraagstuk te tackelen. Bedenk wel dat je in een team werkt met gezamelijk veel kennis. Probeer voor je zelf een limiet te stellen en niet eindeloos te gaan zoeken.

Minimalistisch ontwerp

Bij het ontwikkelen van een systeem, applicatie of code overbodige complexiteit vermijden.
Minimalistische code schrijven gaat verder dan het uitvoeren van een taak door een computer. Mensen moeten deze code onderhouden/uitbreiden en kunnen beschrijven.
