DTSTTCPW

Do The Simplest Thing That Could Possibly Work
Het principe dat ontwikkelaars zich moeten richten op huidige vereisten en niet moet speculeren op toekomstige behoeften. Onderdeel van DTSTTCPW ().

  1. Focus op eenvoud
  2. Iteratieve verbeteringen
  3. Vermijd speculatie
  4. Snelle feedback
  5. Minimalistisch ontwerp

Focus op eenvoud

Bij het oplossen van een programmeer vraagstuk probeer de focus te leggen op eenvoud. Niet de korste of makkelijkste code maar de gemakkelijkst te begrijpen en te onderhouden. Probeer de code te zien vanuit het perspectief van je college of je toekomstige zelf. Kun je de code in een oogopslag doorgronden? Begrijp je wat hier gebeurt?
