Naming conventions

Binnen C# bestaan er geen strikte 'regels' voor de benaming van elementen.
Er zijn echter wel algemeen geaccepteerde richtlijnen voor hoe je dingen benoemt.

  • PascalCase voor Classes, Methods en constants.
  • camelCase voor local variables en method arguments.
  • _camelCase voor private fields
  • I interface prefixes


namespace CleanCode;

public interface INamingConventions
{}

public class NamingConventions: INamingConventions
{
    private string _localValue;

    private const string DefaultValue = "bla";
    
    public string Value { get; set; }
    
    public string GetLocalValue()
    => _localValue;
    
    public void SetLocalValue(string value)
    {
        _localValue = value ?? DefaultValue;
    }
}
