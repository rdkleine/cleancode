Boy Scout Rule

Als een goed padvinder programmeur betaamt, laat je de kampeerplaats codefile schoner achter dan je hem aantrof.
Belangrijke aspecten binnen dit code principe:

  • Continuous Inprovement Wanneer je toch bezig bent in een codefile en je ziet iets wat afwijkt van de coding guidelines pak het dan gelijk mee. Ook kleine verbeteringen - naamgeving, verkeerd gebruik public/private - zijn verbeteringen.
  • Refactoring Vaak is refactoring noodzakelijk om code goed/beter te laten werken of (voor een mens) leesbaar te maken.
  • Voorkom "coderot" Code waar vaak aan gewerkt wordt zal door de loop van de tijd degraderen in leesbaarheid. Door niet alleen te fixen wat kapot is maar ook kijkt naar het geheel voorkom je degradatie.
  • Stimuleren van Good Practices Door te werken aan betere code stimuleer je de adaptatie van good coding practices and standards.
5. **Cultural Impact**: Beyond individual practices, the Boy Scout Rule promotes a culture of responsibility and care among the development team. It fosters an environment where quality is everyone's responsibility. 6. **Not Just Code Cleanup**: It's not only about cleaning up the code. It's also about improving documentation, tests, and operational aspects. 7. **Balancing Act**: While following this rule, it's essential to balance the desire to improve the code with the need to deliver functional software in a timely manner. It's not about making large-scale changes or rewrites, but about incremental, sustainable improvements. In essence, the Boy Scout Rule in programming is about taking small, proactive steps to make the codebase better, healthier, and more sustainable with each contribution.

Leave the code cleaner than you found it.
~ (uncle Bob) Robert C. Martin - From "Clean Code: A Handbook of Agile Software Craftsmanship"
